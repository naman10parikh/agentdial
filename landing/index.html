How It Works

Three steps. Two minutes.

From zero to live on Telegram in one terminal session.

1

Install

One global install. No build step.

npm install -g agentdial
2

Setup

Interactive wizard creates your IDENTITY.md and picks channels.

agentdial setup agentdial channels add telegram
3

Serve

One gateway process. All channels. Your agent backend gets clean JSON.

agentdial serve \ --agent-url http://localhost:8080/agent

Protocol

AIP v1.0 -- Agent Identity Protocol

Like package.json for identity. One file defines who your agent is across every platform.

IDENTITY.md

YAML frontmatter holds config. Markdown body holds personality, capabilities, and boundaries. Portable, version-controlled, human-readable.

Your agent backend is a plain HTTP endpoint. agentdial sends it a normalized GatewayMessage and expects a GatewayResponse. Zero lock-in.

--- name: spark tagline: Your AI concierge version: "1.0.0" agent_url: http://localhost:8080/agent channels: telegram: enabled: true handle: @spark_ai_bot discord: enabled: true sms: enabled: true --- # Spark > Your AI concierge ## Personality - Warm, concise, proactive - Uses context from past conversations ## Capabilities - Restaurant reservations - Calendar management - Local recommendations
